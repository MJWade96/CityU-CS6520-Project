1. **1st Interim Report** [5% marks]  
The 1st Interim Report should discuss your current progress on the study, and should consist of 15 pages. Here are the requirements:  
(1) Introduction – Clearly describe background of the selected topic and the objectives and why the topic is useful and potential outcome (prototype of application software, performance analysis or theoretical results, etc.).  
(2) Related work – Illustrate the problem as associated with existing and related systems/solutions. Description of advantages and disadvantages between existing solutions. Discussion of the relationship between the existing solutions and the proposed solution.  
(3) System modeling and structure – the proposed solution. Discussion of the justifications of the design choices, and how they address existing limitations.  
(4) The methodology and algorithms you will design/implement for the proposed solution.  
(5) Preliminary performance analysis or experiments of your algorithm/system.  
(6) Milestones and overall schedule for your project.  
(7) Work to be completed for the next report.  
(8) References.

Also note the following requirements:  
• Submit a soft copy to course page on Canvas.  
• All figures and table that are taken from other sources need to be cited, e.g., with the following text “(figure from [1])”.  
For case of similarity rate higher than 25%, it may lead to severe penalties.

2. **2nd Interim Report** [5% marks]  
The 2nd interim report should be an expanded version of the 1st Interim Report, consisting of 20-30 pages. Sections (4) (5), and (6) should be expanded with more information, and Section (7) should be updated.

Also note the following requirements:  
• Submit a soft copy to course page on Canvas.  
• All figures and table that are taken from other sources need to be cited, e.g., with the following text “(figure from [1])”.  
The maximum allowable overlap between the two interim reports is 50%. The 2nd interim report should include 50% of new content. If the similarity rate exceeds this threshold, it may result in severe penalties.

3. **Final Report** [20% marks for the report, 46% marks for technical merit]  
The final report should include all the outcomes of your project. It will be assessed on both the technical merit (46% marks) of the report, as well as the quality of the report (20%).  
The final report should be ~30-40 pages (excluding programs).  
The maximum allowable overlap between the two interim reports and the final report is 50%. The final report should include 50% of new content. If the similarity rate exceeds this threshold, it may result in severe penalties.

Here are the requirements:  
(1) Introduction (see description for Interim Reports).  
(2) Related work (see description for Interim Reports).  
(3) System modeling and structure (see description for Interim Reports).  
(4) The methodology and algorithms used in the design/implementation of the system. (see description for Interim Reports).  
(5) Performance analysis or experiments of your algorithm/system – Evaluations comparing to existing works, and results supporting the design choices versus alternative choices.  
(6) Conclusion – A concise summary of the project.  
(7) Future Work – Discussion of possible future directions of work.  
(8) References  

Also note the following requirements:  
• Submit a soft copy to course page on Canvas.  
• Submit source code and project files to Canvas.  
• All figures and tables that are taken from other sources need to be cited, e.g., with the following text “(figure from [1])”.

4. **Presentation** [14% marks]  
The oral presentation should be ‘live’. You should present in person or on Zoom for about 15 minutes, followed by an additional 3–5 minutes for questions from your Supervisor and 2nd Reader. If it is difficult to schedule a presentation time with everyone you can present twice – to the Supervisor and 2nd Reader separately.
5. **Report Format**  
Cover Page: Student name, student number, project title and supervisor name should be written on a cover page of each report.  
Margins (top/bottom/left/right): 2.54 cm  
Font Name: Times New Roman (font size: 12)  
Line Spacing: Single Line

Grading Remarks: As Project is a two-semester research-related course, it is expected that there should be a thorough literature review, analysis of current systems, a proposed system or proposed methodology, substantial testing and experiments, and analysis. Only projects that are complete enough to be submitted to conference or journals will be considered for A or A+ grades.
