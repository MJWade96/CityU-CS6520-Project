### 1. 核心语料库 (The Corpus)
**策略：** 采用 **“教科书级知识 + 科研级补充”** 的组合，确保覆盖绝大多数医学常识题。

| 数据源 | 处理方式 (Chunking) |
| :--- | :--- |
| **StatPearls** | **段落级切分**：按 Markdown 标题或句号切分，控制在 **256-512 token** 之间。 |
| **PubMed Abstracts** | **整摘要作为一个 Chunk**：因为摘要通常较短，直接作为独立单元。 |


### 2. 评估数据集 (The Evaluation Set)
+ **主数据集：MedQA-US (USMLE)**
    - **划分建议：**
        * **开发集 (Dev Set)：** 随机抽取 **300 题**（用于调试超参数，如 top-k）。
        * **测试集 (Test Set)：** 剩余 **~1000 题**（用于最终展示模型效果）。
    - _注：如果觉得 1000 题太多，保留 300-500 题做最终测试也是符合硕士论文标准的。_

### 3. 检索与生成配置 (The RAG Pipeline)
**策略：** 保持简洁，重点在于验证 RAG 是否有效，而非堆砌复杂技术。

+ **检索器 (Retriever)：**
    - **模型：** `BGE (BAAI General Embedding)` 系列（如 `bge-large-en-v1.5`），这是目前中文和英文 RAG 效果最好的开源 Embedding 模型之一。
    - **Top-k：** 建议设置为 **5 或 10**（即检索出 5-10 段最相关的文本给模型看）。
+ **Prompt 模板：**
    - 采用标准的 RAG 模板：

> "Given the following context: {retrieved_contexts}. Answer the question: {question}. Choose from A, B, C, D."
>

### 4. 最终评估指标 (The Metrics)
#### (1) 核心指标 (必须展示)
+ **Accuracy (准确率)：** 这是选择题的命根子。计算公式：`(正确回答的题目数 / 总题目数)`。

#### (2) 检索质量指标 (加分项)
+ **Recall@k (R@5/R@10)：** 计算在检索回来的 Top-5 或 Top-10 的文档中，是否包含了支持正确答案的“黄金段落”。
    - _意义：证明你的检索器找对了地方。_
